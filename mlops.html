STEP 12

MLOps (Machine Learning Ops)

AI 모델의 개발, 배포, 운영을 자동화하여 '지속 가능한 AI 서비스' 구축하기.

왜 MLOps가 필요한가?

전통적인 소프트웨어와 달리 AI 서비스는 **코드 + 데이터 + 모델**이라는 세 가지 요소가 얽혀 있습니다. 모델은 시간이 지나면 성능이 떨어지며(Drift), 데이터를 재학습시키고 다시 배포하는 과정이 반복되어야 합니다.

MLOps는 모델 학습부터 서빙, 모니터링까지의 전 과정을 자동화하여 데이터 과학자와 엔지니어 사이의 장벽을 허물고 프로덕션급 AI 인프라를 제공합니다.

5단계 전문 학습 로드맵

  • 1. ML 라이프사이클과 DevOps

    전통적인 DevOps와 MLOps의 차이점, 그리고 CT(Continuous Training)의 개념을 이해합니다.

  • 2. 데이터 & 모델 버전 관리 (DVC)

    Git으로는 관리하기 힘든 대용량 데이터와 바이너리 모델 파일을 버전 관리하는 기법을 배웁니다.

  • 3. 실험 추적 (MLflow)

    수많은 하이퍼파라미터 실험 결과와 메트릭을 기록하고 최적의 모델을 선정하는 자동화 저장소를 구축합니다.

  • 4. 모델 서빙 & 파이프라인 (Kubeflow)

    학습된 모델을 API 형태로 배포하고, 데이터 전처리부터 학습까지의 파이프라인을 오케스트레이션합니다.

  • 5. 모델 모니터링 & 드리프트 감지

    실제 운영 환경에서 모델의 예측 성능이 떨어지는지 감지하고, 자동으로 재학습을 트리거하는 체계를 설계합니다.
