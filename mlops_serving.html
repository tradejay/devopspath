모델 서빙 & 파이프라인

학습된 모델을 실제 서비스에 연결하는 고성능 배포 아키텍처

1. 모델 서빙 (Serving) 방식

배포된 모델이 어떻게 요청을 처리할 것인가에 따라 두 가지로 나뉩니다.

  • Online Serving: 실시간 REST/gRPC API로 요청에 즉각 응답. (예: 추천 시스템)
  • Batch Serving: 수백만 건의 데이터를 미리 계산하여 데이터베이스에 저장. (예: 대량 뉴스 분류)

Kubeflow Pipelines

ML 워크플로우를 구성하는 여러 단계(데이터 정제 -> 학습 -> 검증 -> 서빙)를 하나의 파이프라인으로 엮어서 관리합니다. 각 단계는 독립적인 컨테이너로 실행되어 확장성이 뛰어납니다.

2. 고성능 서빙 프레임워크

  • BentoML: 파이썬 모델을 쉽고 빠르게 프로덕션급 API 서버로 패키징.
  • Triton Inference Server: GPU 활용을 극대화하여 초당 처리량을 높이는 엔비디아의 솔루션.
  • Seldon Core: K8s 환경에서 고급 배포 전략(Canary, AB Test)을 지원하는 ML 서빙 엔진.

실전 체크포인트: 인프라 오토스케일링

ML 서빙은 CPU/GPU 소모가 매우 큽니다. 요청이 없을 때는 노드를 0으로 줄이고(Serverless), 트래픽이 몰릴 때만 급격히 늘리는 최적화가 필수적입니다.
