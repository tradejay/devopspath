Security Incident Response

사고 발생 시 당황하지 않고 자동화된 절차에 따라 신속하게 대응하기

1. 보안 사고 대응의 기술화

보안 사고가 터졌을 때 사람이 직접 리소스를 끄거나 방화벽을 수정하는 것은 느리고 실수가 많습니다. 사고 대응(Incident Response) 전 과정을 자동화된 시나리오(Playbook)로 구축해야 합니다.

SOAR (Security Orchestration, Automation, and Response)

다양한 보안 도구에서 올라오는 경고를 분석하고, 정해진 워크플로우에 따라 자동으로 대응하는 시스템입니다. 예를 들어 "DDoS 공격 감지 시 해당 IP 즉시 차단" 같은 로직을 자동화합니다.

2. 실무적인 대응 절차 (Step-by-Step)

  1. Detection (탐지): 이상 징후를 실시간으로 포착합니다. (예: 평소보다 100배 높은 로그인 실패)
  2. Containment (봉쇄): 사고 범위를 줄입니다. (예: 해당 컨테이너 즉시 중지, 네트워크 격리)
  3. Eradication (제거): 공격의 근원을 제거하고 보안 취약점을 패치합니다.
  4. Recovery (복구): 깨끗함이 증명된 최신 백업본으로 시스템을 복원합니다.

사후 검토: Post-mortem

사고가 해결된 후 반드시 "무엇이 잘못되었는가?", "어떻게 하면 다음에는 자동으로 막을 수 있는가?"를 기록하세요. 기술적인 패치뿐만 아니라 자동화 프로세스를 개선하는 기회로 삼아야 합니다.
