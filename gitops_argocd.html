ArgoCD 실무 패턴

Kubernetes 전용 GitOps 엔진으로 구현하는 선언적 배포

1. ArgoCD란?

ArgoCD는 Git 저장소의 상태와 클러스터의 상태를 동기화해주는 K8s 컨트롤러입니다.

주요 기능: Sync & Drift Detection

누가 수동으로 클러스터를 수정하면 'OutOfSync' 경고를 띄우거나, 자동으로 원래 상태로 되돌리는(Auto-Heal) 기능을 제공합니다.

2. ApplicationSet: 대규모 관리의 핵심

수백 개의 마이크로서비스를 개별 Application으로 등록하는 것은 비효율적입니다. ApplicationSet을 사용하면 하나의 템플릿으로 여러 클러스터와 네임스페이스에 동적 배포가 가능합니다.

apiVersion: argoproj.io/v1alpha1
kind: ApplicationSet
metadata:
  name: guestbook
spec:
  generators:
  - list:
      elements:
      - cluster: engineering-dev
        url: https://kubernetes.default.svc
      - cluster: engineering-prod
        url: https://1.2.3.4
  template:
    metadata:
      name: '{{cluster}}-guestbook'
    spec:
      project: default
      source:
        repoURL: https://github.com/argoproj/argocd-example-apps.git
        targetRevision: HEAD
        path: guestbook
      destination:
        server: '{{url}}'
        namespace: guestbook

실전 체크리스트

  • Auto-Sync 설정: 자동 동기화 여부를 신중히 결정 (운영은 수동 승인 권장).
  • Prune Resources: Git에서 삭제된 리소스를 클러스터에서도 삭제할 것인가?
  • Self Heal: 수동 수정을 허용할 것인가?
