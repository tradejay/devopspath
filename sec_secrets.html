Secrets Management (비밀 정보 관리)

코드에서 비밀번호를 제거하고 안전한 중앙 저장소 구축하기

1. 저장소에 비밀을 남기지 마세요

Git 저장소에 커밋된 API 키나 DB 비밀번호는 삭제하더라도 히스토리에 남게 되어 매우 위험합니다. 모든 비밀 정보(Secrets)는 코드와 완전히 분리되어야 합니다.

HashiCorp Vault

단순한 저장소를 넘어, 동적 비밀번호 생성(Dynamic Secrets), 데이터 암호화, 세밀한 접근 제어를 제공하는 기업용 보안 엔진입니다. 정적 비밀번호를 수동으로 관리하는 대신 Vault가 주기적으로 비밀번호를 바꾸도록 자동화할 수 있습니다.

2. 비밀 정보 관리의 대안들

  • GitHub Secrets: CI/CD 파이프라인에서만 필요한 비밀 정보를 저장할 때 매우 편리합니다.
  • AWS Secrets Manager: AWS 리소스와 밀접하게 연동되어 RDS 비밀번호 등을 자동으로 순환(Rotation)시킵니다.
  • Environment Variables: 가장 기본적인 방식이지만, 인쇄(print)나 로깅 시 유출되지 않게 주의해야 합니다.
# Vault CLI를 이용한 데이터 읽기 예시
vault kv get -format=json secret/my-app/config | jq .data.data.api_key

# 앱 실행 시 주입 (Example)
export API_KEY=$(vault kv get -field=api_key secret/my-app/config)
npm start

실무 조언: "Never Hardcode"

로컬 개발 시에도 `.env` 파일을 `.gitignore`에 반드시 포함하여 실수로 푸시되지 않도록 습관을 들이세요. `git-secrets` 같은 도구를 설치하면 푸시 전에 비밀 정보 포함 여부를 자동으로 검사해줍니다.
