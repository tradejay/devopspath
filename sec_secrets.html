Secrets Management: 시크릿 관리

패스워드와 API 키를 코드에서 분리하고 금고에 보관하기

1. 시크릿(Secrets)이란?

데이터베이스 비밀번호, 클라우드 API 키, SSH 개인 키 등 외부에 유출되면 서비스에 치명적인 영향을 주는 정보를 말합니다. 이런 정보들은 **절대** 소스 코드나 깃허브 저장소에 포함되어서는 안 됩니다.

사고 발생 시: 즉시 순환(Rotation)

만약 실수로 API 키가 깃허브에 올라갔다면, 파일을 삭제하는 것으로 끝나지 않습니다. 해당 키를 즉시 폐기하고 새로운 키를 발급(Rotation)받아야 합니다.

2. 전문적인 시크릿 보관소

  • HashiCorp Vault: 가장 강력한 범용 시크릿 관리 도구. 동적 키 발급 기능을 제공합니다.
  • AWS Secrets Manager: AWS 서비스와 밀접하게 연동되며, DB 패스워드 자동 변경 기능을 제공합니다.
  • GitHub Secrets: CI/CD 파이프라인에서만 필요한 정보를 보관할 때 유용합니다.

3. 주입 방식 (Dynamic Injection)

애플리케이션은 실행되는 시점에 API를 통해 시크릿 보관소에서 정보를 가져옵니다. `.env` 파일조차 쓰지 않고 환경 변수로 직접 주입하는 방식을 통해, 서버 파일 시스템 어디에도 비밀번호를 남기지 않는 것이 가장 안전합니다.
