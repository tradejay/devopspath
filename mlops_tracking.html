실험 추적 (MLflow)

수천 번의 모델 학습 이력을 일관된 포맷으로 기록하고 관리하기

1. 왜 실험 추적이 필요한가?

모델 성능을 올리기 위해 데이터 과학자는 끊임없이 하이퍼파라미터를 바꿉니다. 이를 엑셀이나 메모장에 적는 것은 한계가 있으며, 나중에 '어떤 설정일 때 가장 잘 나왔는지' 찾기 힘듭니다.

MLflow Tracking

코드 한 줄로 학습 시 사용한 파라미터, 성능 지표(Accuracy, Loss), 손실 그래프 등을 중앙 서버에 자동으로 기록합니다.

2. 핵심 기능 3가지

  • Tracking: 코드 실행 결과 기록 및 시각화.
  • Projects: 재사용 가능한 데이터 과학 코드 패키징.
  • Models: 다양한 라이브러리(PyTorch, TensorFlow 등)로 만든 모델을 표준 형식으로 저장.
# MLflow 코드 예시
import mlflow

with mlflow.start_run():
    mlflow.log_param("learning_rate", 0.01)
    mlflow.log_metric("accuracy", 0.95)
    mlflow.log_artifact("model.h5") # 실제 모델 파일 저장

운영 팁: 대시보드 활용

MLflow 대시보드에서 여러 실험들을 한눈에 비교(Compare)하고, 성능이 가장 좋은 가중치를 클릭 한 번으로 '운영(Production)' 단계로 승격시킬 수 있습니다.
