K8s 위에서의 CI/CD

Helm 차트와 ArgoCD를 이용한 현대적인 지속적 배포 전략

1. Helm 차트: K8s 패키지 관리

수많은 YAML 파일을 하나하나 관리하는 대신, 템플릿화된 **Helm**을 통해 버전 관리와 재사용성을 높입니다.

왜 Helm인가?

환경별(Dev/Prod)로 다른 설정값을 values.yaml 하나로 통제할 수 있으며, 배포 실패 시 `helm rollback`으로 즉시 복구가 가능합니다.

2. GitOps와 ArgoCD

GitPush로 배포를 실행하는 기존 방식에서 벗어나, K8s 클러스터가 Git의 상태를 감시하고 자신의 상태를 일치시키는 **Pull 방식**의 배포를 지향합니다.

# ArgoCD를 통한 배포 자동화 흐름
1. 개발자가 코드를 Git에 Push
2. CI(GitHub Actions)가 이미지를 빌드하고 레지스트리에 Push
3. CI가 매니페스트(Helm) 저장소의 버전 정보를 업데이트
4. ArgoCD가 저장소 변경을 감지하고 클러스터에 Sync 실행

GitOps의 이점

  • 클러스터 자격증명을 CI 서버에 넘길 필요가 없어 보안상 우수함.
  • Git 히스토리가 곧 인프라의 히스토리가 됨 (Audit Trail).
  • 사람의 실수(수동 kubectl 명령)를 원천 차단함.
