Docker Foundations (컨테이너 원리)

가상 머신과는 다른, 컨테이너만의 효율적인 격리 기술 이해하기

1. 컨테이너 vs 가상 머신(VM)

컨테이너는 OS 전체를 가상화하는 것이 아니라, 호스트의 커널을 공유하며 프로세스 레벨에서 격리합니다.

  • VM: 하드웨어 수준 가상화. 무겁고 부팅이 느림.
  • Container: OS 수준 가상화. 가볍고 부팅이 매우 빠름.

격리의 핵심 기술

  • Namespaces: 프로세스별로 독립적인 공간(PID, Network, Mount 등)을 할당하여 서로 못 보게 함.
  • Control Groups (Cgroups): 프로세스가 사용할 리소스(CPU, 메모리 등)의 양을 제한함.

2. Docker 아키텍처

Docker는 클라이언트-서버 모델을 따릅니다. 사용자가 명령을 내리면 Docker Daemon이 이를 처리하고 컨테이너를 관리합니다.

# 도식화: 운영 원리
[ Client (CLI) ] --- (API) ---> [ Docker Daemon (Host) ]
                                      |
                            -------------------------
                            |          |            |
                       [ Image ]   [ Container ]  [ Registry ]

전문가의 팁: "컨테이너는 프로세스다"

컨테이너 내부에서 SSH 서버를 실행하거나 여러 개의 앱을 돌리려 하지 마세요. 하나의 컨테이너는 하나의 책임(프로세스)만 가져야 관리와 확장이 쉽습니다.
