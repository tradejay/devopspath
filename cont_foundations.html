Foundations: VM vs Container

격리의 수준과 하이퍼바이저와 도커 엔진의 차이

1. 가상머신(VM)의 한계

전통적인 가상머신은 하드웨어 위에 하이퍼바이저를 올리고, 그 위에 **게스트 OS**를 통째로 설치합니다. 이 방식은 격리 수준은 높지만, 부팅 속도가 느리고 수 기가바이트의 메모리를 기본으로 점유한다는 단점이 있습니다.

컨테이너의 핵심: 커널 공유

컨테이너는 호스트 OS의 커널(Kernel)을 공유하며, 프로세스 레벨에서 필요한 라이브러리만 격리하여 실행합니다. 덕분에 매우 가볍고(MB 단위), 1초 안에 실행될 수 있습니다.

2. Docker 엔진의 구조

도커는 다음과 같은 구성 요소로 움직입니다.

  • Docker Client: 사용자가 명령을 내리는 인터페이스 (CLI).
  • Docker Host (Daemon): 컨테이너를 빌드하고 실행하는 실제 엔진.
  • Registry: 만들어진 이미지를 저장하고 배포하는 저장소 (Docker Hub).

3. 실습 가이드

터미널에서 docker info 명령어를 쳐보세요. 현재 본인의 컴퓨터에서 돌아가는 도커 엔진이 어떤 커널 정보를 사용하는지 확인할 수 있습니다.
