Infrastructure Pipeline: 인프라 자동화

인프라 코드의 리뷰와 배포를 자동화하는 CI/CD 워크플로우

1. 인프라 배포의 위험성

로컬 터미널에서 `terraform apply`를 직접 치는 것은 위험합니다. 누가 어떤 리소스를 바꿨는지 팀원들이 알기 어렵고, 실수로 잘못된 환경(예: 운영 환경)에 배포할 수도 있기 때문입니다.

2. 인프라 CI/CD 워크플로우

전문적인 환경에서는 인프라 변경도 애플리케이션 코드처럼 **풀 리퀘스트(PR)** 과정을 거칩니다.

  • Plan Step: PR이 올라오면 자동으로 `terraform plan`을 실행하여 변경될 내용을 댓글로 남깁니다.
  • Review Step: 동료 엔지니어가 예상 변경 내용을 검토하고 승인합니다.
  • Apply Step: 메인 브랜치에 코드가 병합되면 실제로 인프라에 반영합니다.
# GitHub Actions 예시
steps:
  - name: Terraform Plan
    run: terraform plan -out=tfplan
    
  - name: Terraform Apply
    if: github.ref == 'refs/heads/main'
    run: terraform apply tfplan

3. 보안 도구 연동

파이프라인 단계에서 **tfsec**이나 **Checkov** 같은 도구를 연동하면, 코드를 배포하기 전에 "실수로 S3 버킷을 전체 공개로 설정했는지" 같은 보안 취약점을 자동으로 찾아낼 수 있습니다.
