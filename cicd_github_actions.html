Foundations & CI (지속적 통합)

모든 것을 코드화하고, 모든 커밋에 대해 자동으로 빌드 및 검토하기

1. Foundations: Version Control Everything

현대적 CI/CD의 첫 번째 원칙은 **"저장소에 없는 것은 존재하지 않는 것"**입니다. 소스 코드뿐만 아니라 DB 스키마, 서버 설정(Config), 인프라 선언문(IaC), 빌드 스크립트 등 서비스 운영에 필요한 모든 자산이 Git과 같은 버전 관리 시스템에서 관리되어야 합니다.

전문가의 브랜칭 전략: Trunk-based Development

전문 서적들은 '짧은 수명의 브랜치'를 권장합니다. 복잡한 GitFlow보다는 메인 브랜치(Trunk)에 최소 하루에 한 번 이상 병합하여 '통합의 고통'을 줄이는 것이 CI의 핵심입니다.

2. Continuous Integration (CI): 피드백의 속도

CI는 단순히 '자동 배포'가 아닙니다. 핵심은 **"내가 짠 코드가 전체 시스템을 망가뜨리지 않았는가?"**에 대한 답을 몇 분 안에 듣는 것입니다. 이를 위해 다음과 같은 과정이 매 커밋마다 수행되어야 합니다.

  • 컴파일 및 빌드: 문법 오류가 없는지, 빌드가 성공하는지 확인.
  • 정적 분석(Linting): 팀의 코드 컨벤션을 준수하는지 자동 검사.
  • 보안 스캐닝: 종속성 라이브러리에 취약점이 없는지 확인.

3. 통합 워크플로우 예시 (YAML)

기초(Foundations)와 CI를 하나의 파이프라인으로 구성한 예시입니다.

name: Core CI Pipeline

on:
  push:
    branches: [ "main" ]

jobs:
  verify:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      
      # 1. 환경 구성
      - name: Setup Environment
        uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'npm' # 성능 고도화를 위한 캐싱 활용

      # 2. CI 검증 단계
      - name: Install & Lint
        run: |
          npm ci
          npm run lint
          
      # 3. 빌드 테스트 (Artifact 생성을 위한 준비)
      - name: Build
        run: npm run build
