Foundations & CI (지속적 통합)

변경 사항을 두려워하지 않는 개발 문화를 위한 기반 구축

1. 형상 관리 원칙: Version Control Everything

데브옵스의 첫 번째 규칙은 **"코드가 없는 시스템은 없다"**는 것입니다. 소스 코드뿐만 아니라, 빌드 설정, 배포 스크립트, 심지어 팀의 의사결정 기록까지 모두 버전 관리 도구(Git)에서 관리되어야 합니다.

Trunk-based Development

복잡한 브랜치 전략(GitFlow 등)보다는, 메인 브랜치에 수시로 코드를 통합하여 '통합의 고통'을 0으로 만드는 전략이 현대적 CI의 대세입니다. 브랜치의 생명주기를 1일 이내로 유지하세요.

2. Continuous Integration (CI)의 핵심 요건

단순히 코드를 합치는 것이 CI가 아닙니다. 합쳐진 코드가 **"언제나 배포 가능한 상태"**임을 증명해야 합니다.

  • Automated Build: 어떤 환경에서도 명령 한 줄로 빌드가 가능해야 합니다.
  • Self-Verifying: 사람이 눈으로 확인하는 것이 아니라, 수백 개의 단위 테스트가 자동으로 합격 통보를 내려야 합니다.
  • Fast Feedback: 빌드와 테스트 결과는 10분 내로 개발자에게 전달되어야 합니다.
# 고성능 CI를 위한 GitHub Actions 예시
name: Elite CI Workflow

on:
  push:
    branches: [ main ]
  pull_request:
    branches: [ main ]

jobs:
  build-and-test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      
      - name: Cache dependencies # 성능 최적화의 핵심
        uses: actions/cache@v3
        with:
          path: ~/.npm
          key: ${{ runner.os }}-node-${{ hashFiles('**/package-lock.json') }}
          
      - name: Install & Lint
        run: |
          npm ci
          npm run lint
          
      - name: Run Unit Tests
        run: npm test

전문가의 팁: 캐싱 활용

실제 프로덕션 환경에서는 빌드 시간을 단축하는 것이 비용 절감과 직결됩니다. 의존성 패키지와 빌드 결과를 캐싱하여 피드백 루프를 극대화하세요.
