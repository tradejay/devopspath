Modules & DRY: 인프라 모듈화

복잡한 인프라를 부품화하여 중복 없는 코드로 관리하기

1. 테라폼 모듈이란?

모듈은 관련된 여러 리소스를 하나의 **부품(Component)**으로 묶은 것입니다. 예를 들어 "VPC, 서브넷, 보안 그룹"을 하나의 '네트워크 모듈'로 묶어두면, 프로젝트를 새로 시작할 때마다 이 모듈 하나만 가져다 쓰면 됩니다.

DRY (Don't Repeat Yourself) 원칙

같은 코드를 복사하고 붙여넣는 대신, 변수(Variables)를 통해 설정값만 바꿔서 모듈을 재사용하세요. 이는 유지보수 시간을 획기적으로 줄여줍니다.

2. 모듈의 구성 요소

  • Inputs (variables.tf): 모듈을 호출할 때 넘겨주는 설정값 (예: 서버 대수, 인스턴스 타입).
  • Outputs (outputs.tf): 모듈이 생성한 뒤 결과로 돌려주는 값 (예: 생성된 서버의 IP 주소).
  • Resources (main.tf): 실제로 생성될 AWS 리소스 정의.

3. 실무에서의 활용

전문적인 조직은 검증된 모듈을 **별도의 Git 저장소**나 **Private Registry**에 올려두고 사용합니다. 이를 통해 팀 전체가 보안 규정을 준수하는 동일한 표준 인프라를 즉시 구축할 수 있게 됩니다.
