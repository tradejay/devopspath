Reusable Modules (리소스 모듈화)

복잡한 인프라를 부품화하여 코드의 중복을 제거하고 관리 효율 높이기

1. 왜 모듈화가 필요한가?

서버, 데이터베이스, 네트워크 설정을 매번 새로 작성하는 것은 비효율적이며 실수를 유발합니다. 모듈화는 **"한 번 잘 만든 인프라 부품을 여러 곳에서 재사용"**하는 기술입니다.

Terraform Modules

연관된 리소스 묶음을 하나의 단위(Module)로 패키징합니다. 예를 들어 'VPC 모듈'은 서브넷, 라우팅 테이블, 인터넷 게이트웨이를 한꺼번에 생성해주는 검증된 템플릿 역할을 합니다.

2. 모듈의 구성 요소

  • Variables (Input): 모듈 실행 시 외부에서 주입받는 동적 값 (예: 인스턴스 타입, 리전).
  • Main Logic (Resources): 실제 인프라를 생성하는 핵심 테라폼 코드.
  • Outputs (Return): 생성된 리소스의 정보(ID, 접속 주소 등)를 외부로 반환.
# 모듈 호출 예시
module "vpc" {
  source = "./modules/aws-vpc"
  
  vpc_name   = "production-vpc"
  cidr_block = "10.0.0.0/16"
}

output "vpc_id" {
  value = module.vpc.vpc_id
}

추천 전략: Semantic Versioning

모듈을 공용 저장소(GitHub 레지스트리 등)에 올릴 때는 반드시 태그를 통한 버전 관리를 하세요. 모듈의 코드가 바뀌어도 사용하는 쪽에서 버전을 고정해두면 의도치 않은 인프라 변경을 막을 수 있습니다.
