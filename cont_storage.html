Storage & Persistence (데이터 보존)

휘발성 컨테이너 환경에서 데이터를 반영구적으로 관리하기

1. 컨테이너의 데이터는 '휘발성'입니다

컨테이너 내부에서 생성된 파일은 컨테이너가 삭제되면 함께 사라집니다. 데이터베이스 파일이나 로그 등 보관이 필요한 자산은 호스트 시스템의 저장소에 연결해야 합니다.

Volumes vs Bind Mounts

  • Volumes: Docker가 관리하는 저장 공간. 성능이 좋고 관리가 쉬워 가장 권장됨.
  • Bind Mounts: 호스트의 특정 폴더를 직접 연결. 설정 파일(config)을 주입할 때 주로 사용.
  • tmpfs: 메모리에만 저장. 보안이 중요한 임시 파일 처리에 적합.

2. 볼륨 사용 전략

  • Anonymous Volume: 이름 없는 볼륨. 임시 데이터 처리에 쓰이지만 관리가 어려워 권장하지 않음.
  • Named Volume: 이름이 있는 볼륨. 여러 컨테이너가 공유하거나 나중에 다시 연결하기 좋음.
# 볼륨 생성 및 컨테이너 연결 예시
docker volume create my-db-data

docker run -d \
  --name production-db \
  -v my-db-data:/var/lib/mysql \
  mysql:8.0

실무 조언: 데이터 백업

볼륨을 쓴다고 해서 백업이 공짜로 되는 것은 아닙니다. 주기적으로 `docker run --volumes-from` 등의 명령을 사용하여 볼륨 데이터를 아카이빙하거나, 클라우드의 스냅샷 기능을 연동하세요.
