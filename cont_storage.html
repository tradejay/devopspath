Storage & Persistence: 데이터 보존

컨테이너가 삭제되어도 데이터는 사라지지 않아야 합니다

1. 컨테이너의 비영속성 (Ephemeral)

컨테이너 내부에서 생성된 데이터는 컨테이너가 삭제되면 **함께 사라집니다.** DB의 데이터나 사용자가 업로드한 이미지가 사라지는 비극을 막으려면, 데이터를 컨테이너 외부(호스트 서버)에 저장해야 합니다.

주의: 컨테이너 내부 데이터는 휘발성

도커 로그나 임시 파일은 괜찮지만, 영구적으로 보관해야 할 데이터는 절대 컨테이너 내부에만 두지 마세요.

2. 볼륨(Volume) vs 바인드 마운트(Bind Mount)

  • Volumes: 도커가 직접 관리하는 보관 구역. 성능이 좋고 관리가 쉬워 실무에서 상용 환경에 가장 많이 쓰입니다.
  • Bind Mounts: 호스트의 특정 폴더를 컨테이너와 연결. 개발 환경에서 소스 코드를 바로 반영하고 싶을 때 유용합니다.

3. 보관 전략

데이터베이스 컨테이너를 띄울 때는 반드시 -v db_data:/var/lib/postgresql/data 처럼 명시적인 볼륨 설정을 해야 합니다. 이것이 컨테이너 환경에서 무중단으로 데이터를 안전하게 관리하는 핵심 기술입니다.
