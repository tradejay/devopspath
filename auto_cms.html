Headless CMS Integration

노션(Notion)을 백엔드로 사용하여 누구나 쉽게 콘텐츠를 관리하기

1. 왜 Headless CMS인가?

전통적인 CMS(워드프레스 등)는 화면과 데이터가 결속되어 있어 수정이 어렵습니다. Headless CMS는 **"화면(Head)이 없는 데이터 저장소"**로, API를 통해 데이터를 제공하기만 합니다.

Notion as a CMS

개발자뿐만 아니라 비개발자도 익숙하게 사용하는 Notion을 데이터베이스로 활용할 수 있습니다. Notion API를 통해 문서 내용을 JSON으로 가져와 블로그에 뿌려주는 모델입니다.

2. 데이터 흐름 설계

  • Content Creation: 사용자가 Notion 페이지에서 글을 작성합니다.
  • API Fetch: 빌드 타임에 Notion API가 페이지 목록과 본문(Block) 데이터를 가져옵니다.
  • Parsing: Notion 특유의 데이터 형식을 HTML로 변환합니다.
  • Rendering: Next.js 같은 프레임워크가 변환된 HTML을 정적 페이지로 굽습니다.
# Notion Client 연동 예시
const { Client } = require("@notionhq/client")

const notion = new Client({ auth: process.env.NOTION_KEY })

async function getBlogPosts() {
  const response = await notion.databases.query({
    database_id: process.env.NOTION_DATABASE_ID,
    filter: { property: "Status", select: { equals: "Published" } }
  })
  return response.results
}

실무 팁: 마크다운 변환기 활용

Notion 데이터를 직접 처리하는 것은 복잡하므로, `notion-to-md` 같은 라이브러리를 사용하여 마크다운으로 먼저 변환한 뒤 렌더링하는 소트웨어를 구축하세요. 유지보수가 훨씬 쉬워집니다.
