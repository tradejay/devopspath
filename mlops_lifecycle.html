ML 라이프사이클과 DevOps

데이터가 주도하는 소프트웨어 개발의 새로운 흐름

1. MLOps vs DevOps

전통적인 소프트웨어 제품과 기계 학습 모델 중심의 제품은 수명 주기부터 다릅니다.

  • DevOps: 코드의 변경이 배포를 트리거함. (CI/CD)
  • MLOps: 코드 변경뿐만 아니라 데이터의 변경도 배포를 트리거함. (CI/CD/CT)

CT (Continuous Training)

지속적 학습(CT)은 MLOps만의 고유한 개념입니다. 새로운 데이터가 수집되면 파이프라인이 자동으로 돌아가 모델을 재학습시키고 검증합니다.

2. 3가지 자동화 레벨 (Google 기준)

  1. Level 0: 수동 프로세스. 데이터 추출, 학습, 모델 서빙이 모두 분절됨.
  2. Level 1: ML 파이프라인 자동화. 새로운 데이터를 바탕으로 모델을 자동 학습함.
  3. Level 2: CI/CD 파이프라인 자동화. 실험 단계에서 운영 단계로 가는 과정 자체가 자동화됨.

핵심 가치: 재현성 (Reproducibility)

동일한 코드와 동일한 데이터가 주어졌을 때, 언제 어디서든 동일한 성능의 모델이 만들어질 수 있도록 인프라를 설계해야 합니다.
