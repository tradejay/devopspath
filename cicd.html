STEP 01

CI/CD (지속적 통합/배포) 고도화

자동 배표를 넘어, '품질'과 '신뢰'가 담보된 엔지니어링 파이프라인 구축하기.

단순히 배포하는 것이 목적이 아닙니다

데브옵스에서 CI/CD는 단순한 자동화 툴이 아닙니다. 이것은 **"변경 사항을 사용자에게 전달하는 가장 안전한 길"**을 설계하는 과정입니다. 코드가 병합될 때마다 수백 개의 테스트가 돌아가고, 보안 취약점을 검사하며, 인프라의 정합성을 확인해야 합니다.

본 섹션에서는 GitHub Actions를 활용하여 현대적인 배포 파이프라인의 필수 구성 요소들을 하나씩 실습하며 체득합니다.

5단계 전문 학습 로드맵

  • 1. Foundations & CI

    모든 것을 코드화(Version Control)하고, 빠른 피드백을 위한 자동 빌드/린트 환경을 구축합니다.

  • 2. Testing Strategy

    단위 테스트, 통합 테스트, E2E 테스트를 파이프라인에 녹여내어 '장애를 미리 발견하는 법'을 배웁니다.

  • 3. Pipeline Design

    Reusable Workflows와 Job 간 의존성 설계를 통해 복잡한 파이프라인을 체계적으로 관리합니다.

  • 4. Deployment Strategies

    Blue-Green, Canary 등 리스크를 분산시키는 고급 배포 기법을 실무 수준에서 이해합니다.

  • 5. DevSecOps (Shift-Left)

    코드 스캐닝, 종속성 보안 검사를 파이프라인 초기에 배치하여 보안을 자동화합니다.
