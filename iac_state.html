State Management: 상태 관리

인프라의 생명줄, .tfstate 파일을 다루는 안전한 방법

1. .tfstate 파일이란?

테라폼은 사용자가 작성한 코드와 실제 클라우드 인프라가 어떻게 매핑되어 있는지를 `terraform.tfstate`라는 파일에 기록합니다. 이 파일이 없으면 테라폼은 수많은 리소스 중 어떤 것이 자신이 만든 것인지 알 수 없습니다.

절대 금기: 로컬에 보관하지 마세요!

로컬 PC의 `tfstate` 파일을 잃어버리거나 다른 개발자와 다른 버전을 가지고 있으면 인프라가 엉망이 됩니다. 또한 이 파일에는 패스워드 등 민감한 정보가 포함될 수 있어 보안상 매우 위험합니다.

2. 원격 백엔드 (Remote Backend)

협업을 위해 `tfstate` 파일을 AWS S3 같은 **중앙 저장소**에 보관해야 합니다. 이때, 두 명이 동시에 수정해서 충돌이 나는 것을 막기 위해 **DynamoDB를 이용한 잠금(Locking)** 기능을 함께 사용합니다. 

terraform {
  backend "s3" {
    bucket         = "my-terraform-state"
    key            = "prod/terraform.tfstate"
    region         = "ap-northeast-2"
    dynamodb_table = "terraform-lock"
  }
}

3. 워크스페이스 (Workspaces)

환경(Dev, Staging, Prod)을 분리할 때 유용합니다. 똑같은 코드를 쓰되, 워크스페이스만 바꿔서 별도의 인프라 그룹을 격리하여 관리할 수 있습니다.
