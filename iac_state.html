State Management (상태 관리)

인프라의 과거와 현재를 연결하는 Terraform의 '기억 장치'

1. Terraform State 파일의 정체

Terraform은 코드로 정의된 자산과 실제 클라우드 리소스를 맵핑하기 위해 별도의 `terraform.tfstate` 파일을 유지합니다. 여기에는 실제 리소스의 ID, 설정 정보 등이 JSON 형식으로 저장됩니다.

왜 State가 중요한가?

  • 리소스 추적: 어떤 자산이 테라폼에 의해 관리되는지 식별합니다.
  • 변경 사항 감지: 코드와 실제 환경 사이의 차이점(Diff)을 계산합니다.
  • 메타데이터 관리: 리소스 간의 의존성 정보를 저장합니다.

2. 원격 상태 저장 (Remote State Backend)

협업 시 로컬에 State 파일을 두면 서로 동기화되지 않아 인프라가 꼬이게 됩니다. 반드시 S3, GCS 같은 원격 저장소를 사용해야 합니다.

  • Shared Storage: 팀원 모두가 동일한 State를 참조하게 함.
  • State Locking: 동시에 여러 명이 인프라를 수정하지 못하게 잠금(Ex: DynamoDB) 처리. (Race Condition 방지)
# S3 백엔드 설정 예시
terraform {
  backend "s3" {
    bucket         = "my-terraform-state-bucket"
    key            = "dev/frontend/terraform.tfstate"
    region         = "ap-northeast-2"
    dynamodb_table = "terraform-lock-table"
  }
}

주의: State는 비밀 저장소가 아닙니다

State 파일에는 인프라의 모든 정보가 평문(Plain text)으로 저장됩니다. DB 비밀번호 등의 민감 정보도 포함될 수 있으므로, State 버킷은 절대적으로 접근을 제한하고 암호화해야 합니다.
