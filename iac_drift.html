Drift Detection & Management

코드와 실제 인프라 사이의 간극을 탐지하고 정합성 유지하기

1. 인프라 데리프트(Drift)란 무엇인가?

코드형 인프라(IaC)로 관리되는 시스템에서 누군가가 웹 콘솔을 통해 수동으로 설정을 변경하면, 코드(State)와 실제 리소스 사이에 차이가 발생합니다. 이를 **Drift**라고 합니다.

Drift의 위험성

  • 예측 불가능성: 코드를 실행했을 때 어떤 일이 벌어질지 알 수 없게 됩니다.
  • 복구 불가능: 재난 발생 시 기존 코드로 똑같은 환경을 재구축할 수 없습니다.
  • 보안 홀: 수동으로 열어둔 방화벽 포트가 코드에는 나타나지 않아 방치될 수 있습니다.

2. Drift 탐지 및 수정 프로세스

정기적으로 인프라의 상태를 체크하여 동기화해야 합니다.

  • Scheduled Plan: 매일 아침 자동으로 `terraform plan`을 돌려 변경 사항이 있는지 메신저로 알림을 보냅니다.
  • Refresh: 실제 환경의 정보를 State 파일에 강제로 업데이트합니다.
  • Reconciliation: 실제 환경을 다시 코드에 정의된 상태로 되돌립니다. (가장 권장되는 방식)
# Drift 탐지 명령어
terraform plan -detailed-exitcode

# 0: 변경 없음
# 1: 에러 발생
# 2: Drift 존재 (코드와 실제 환경이 다름)

문화적 해결책: 권한 제어

기술적인 탐지보다 중요한 것은 "수동 변경이 불가능한 구조"를 만드는 것입니다. 엔지니어들에게 클라우드 콘솔의 '읽기 권한'만 부여하고, 모든 변경은 'CI/CD 파이프라인 전용 계정'을 통해서만 일어나도록 강제하세요.
