자동 복구 & 스케일링

Probe를 이용한 자기 치유 환경과 트래픽 기반 자동화

1. Liveness vs Readiness Probe

포드가 살아있는지(Liveness), 그리고 요청을 받을 준비가 되었는지(Readiness)를 구분하여 체크해야 합니다.

자기 치유(Self-Healing)

Liveness Probe가 실패하면 K8s는 해당 컨테이너를 즉시 재시작합니다. 애플리케이션 코드가 데드락에 빠져도 시스템이 스스로 복구합니다.

2. HPA (Horizontal Pod Autoscaler)

트래픽이 몰리면 CPU/메모리 사용량을 감지하여 자동으로 포드의 개수를 늘려줍니다.

apiVersion: autoscaling/v2
kind: HorizontalPodAutoscaler
metadata:
  name: web-hpa
spec:
  scaleTargetRef:
    apiVersion: apps/v1
    kind: Deployment
    name: web-app
  minReplicas: 2
  maxReplicas: 10
  metrics:
  - type: Resource
    resource:
      name: cpu
      target:
        type: Utilization
        averageUtilization: 70

실전 체크포인트

  • Startup Probe: 서비스 부팅 시간이 길 때 초기 재시작 방지.
  • Resource Limits: HPA가 동작하려면 반드시 CPU Limit이 설정되어야 함.
  • Graceful Shutdown: 종료 시 진행 중인 요청을 안전하게 처리할 시간을 확보해야 함.
