데이터 파이프라인 (ETL/ELT)

데이터의 흐름을 설계하고 관리하는 파이프라인 자동화

1. ETL vs ELT

데이터를 어떻게 처리하느냐에 따라 두 가지 모델로 나뉩니다.

  • ETL (Extract -> Transform -> Load): 데이터를 추출한 후, 중간 서버에서 변환하여 적재함. 전통적인 방식.
  • ELT (Extract -> Load -> Transform): 데이터를 먼저 대용량 저장소(DW)에 적재한 후, DW의 강력한 성능을 이용해 변환함. 클라우드 시대의 주류.

Apache Airflow: 워크플로우 오케스트레이션

파이썬 코드로 복잡한 데이터 파이프라인의 의존성(DAG)을 정의합니다. 실패 시 재시도, 스케줄링, 실패 알림 등을 중앙에서 관리합니다.

2. 실시간 vs 배치 처리

  • Batch: 특정 주기(예: 매일 자정)마다 대량의 데이터를 수집. (Apache Spark)
  • Streaming: 발생하는 즉시 데이터를 수집 및 처리. (Apache Kafka, Flink)
# Airflow DAG 예시 (Python)
with DAG('daily_data_sync', schedule_interval='@daily') as dag:
    extract = PythonOperator(task_id='extract', python_callable=extract_from_api)
    transform = SparkSubmitOperator(task_id='transform', application='clean_data.py')
    load = BigQueryInsertJobOperator(task_id='load', configuration={...})

    extract >> transform >> load # 의존성 정의

실전 팁: 멱등성 (Idempotency)

동일한 파이프라인을 여러 번 실행해도 결과 데이터가 중복되거나 깨지지 않도록 '기존 데이터 삭제 후 재적재'와 같은 멱등성이 보장되게 설계해야 합니다.
