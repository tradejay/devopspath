데이터 & 모델 버전 관리

Git의 한계를 넘는 대용량 AI 자산 관리 표준

1. 왜 Git만으로는 안 되는가?

Git은 텍스트 파일의 미세한 변화를 추적하는 데 최적화되어 있습니다. 수십 GB의 데이터셋이나 바이너리 모델 파일을 Git에 올리면 저장소가 무거워지고 성능이 급격히 저하됩니다.

DVC (Data Version Control)

Git과 같은 방식으로 작동하지만, 실제 데이터는 S3나 Google Cloud Storage 같은 외부 저장소에 두고 Git에는 해당 데이터를 가리키는 '메타데이터(포인터 파일)'만 저장합니다.

2. 데이터 버전 관리의 이점

  • 실험 재현: 특정 시점의 모델이 어떤 데이터로 학습되었는지 완벽하게 추적 가능.
  • 효율적인 공유: 큰 파일을 주고받을 필요 없이 포인터 파일만 공유하고 `dvc pull`로 데이터 획득.
  • 워크플로우 통합: 데이터 변경 시 자동으로 학습 파이프라인을 트리거하는 기초가 됨.
# DVC 기본 명령어
dvc add data/original_dataset.csv  # 데이터 추가 (포인터 파일 생성)
git add data/original_dataset.csv.dvc .gitignore
git commit -m "Add raw data version 1"
dvc push  # 실제 데이터를 원격 저장소(S3 등)로 전송

꿀팁: 모델 레지스트리

단순한 파일 관리를 넘어, 테스트를 통과한 '승인된 모델'만 버전별로 관리하는 모델 레지스트리(Model Registry) 개념을 함께 도입해야 합니다.
