제로 트러스트 보안 (mTLS)

클러스터 내부 통신의 암호화와 강력한 인증 체계 구축

1. mTLS (Mutual TLS)란?

클라이언트와 서버가 서로를 확인하는 상호 인증 방식입니다. Istio를 사용하면 애플리케이션 수정 없이 서비스 간 모든 트래픽을 자동으로 mTLS 암호화할 수 있습니다.

왜 필요한가?

클러스터 네트워크에 침입자가 발생하더라도, 인증서가 없는 통신은 즉시 차단되어 2차 피해(Lateral Movement)를 방지할 수 있습니다.

2. AuthorizationPolicy (인가)

어떤 서비스가 어떤 서비스에게, 어떤 경로(/api/...)로 접근할 수 있는지 선언적으로 정의합니다.

# 인증 정책 예시 (mTLS 강제)
apiVersion: security.istio.io/v1beta1
kind: PeerAuthentication
metadata:
  name: default
spec:
  mtls:
    mode: STRICT

보안 체크포인트

  • STRICT Mode: 암호화되지 않은 요청은 원천 차단.
  • JWT 연동: 외부 요청에 대한 인증을 사이드카 레벨에서 처리.
  • RBAC: 서비스 간 최소 권한 원칙(Least Privilege) 준수.
