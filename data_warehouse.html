분석 매체 (Warehouse vs Lakehouse)

데이터의 규모와 성격에 최적화된 저장공간 선택하기

1. Modern Data Storage 비교

구분 특징
Data Warehouse 정형 데이터 중심. 초고성능 SQL 분석 최적화. (Snowflake, BigQuery)
Data Lake 비정형 데이터 포함. 저렴한 비용으로 대량 저장. (S3 + Athena)
Data Lakehouse 레이크의 저렴한 비용 + 웨어하우스의 관리 기능 통합. (Databricks, Iceberg)

dbt (data build tool): 분석가도 엔지니어처럼

SQL만으로 데이터 변환(Transform) 과정을 작성하고, 이를 버전 관리 및 문서화할 수 있게 돕는 도구입니다. '데이터 분석의 엔지니어링화'를 주도하고 있습니다.

2. 대용량 쿼리 최적화 기법

  • Partitioning: 날짜나 키 값으로 데이터를 물리적으로 쪼개어 스캔 범위를 줄임.
  • Clustering: 자주 같이 조회되는 컬럼끼리 모아서 저장.
  • Columnar Storage: 행이 아닌 열(Column) 단위로 저장하여 분석 쿼리 속도 극대화. (Snappy, Parquet)

실무 조언: 인프라 분어

데이터 웨어하우스는 컴퓨팅 비용이 비쌉니다. 무분별한 `SELECT *`를 막고, 쿼리 비용을 실시간으로 태깅하여 팀별로 비용을 배분(Showback)하는 것이 핀옵스와 연결되는 지점입니다.
