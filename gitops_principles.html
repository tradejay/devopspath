GitOps 4대 핵심 원칙

Git을 단순한 저장소가 아닌 '인프라의 뇌'로 만드는 방법

1. 선언적 성능 (Declarative)

모든 시스템의 상태가 '명령어'가 아닌 '상태 정의 파일'로 존재해야 합니다.

원칙: 진실의 원천(SSOT)

Git 저장소에 있는 코드가 곧 현재 가동 중인 시스템의 모습과 100% 일치해야 합니다.

2. 4가지 원칙 상세

  • 버전 관리 및 불변성: 모든 변경 이력은 Git에 남으며 되돌리기가 쉬워야 합니다.
  • 자동 당김 (Pull-based Sync): 클러스터 내부의 에이전트가 Git의 변화를 감시합니다.
  • 지속적 조정 (Continuous Reconciliation): 실제 상태와 원하는 상태가 다르면 즉시 원복합니다.
  • 선언적 시스템 상태: 시스템은 오직 선언된 명세에 의해서만 움직입니다.

고비용 실수: Git 직접 수정 금지

운영 환경에 kubectl로 직접 설정값을 바꾸면 GitOps 에이전트가 이를 '구성 드리프트(Drift)'로 간주하고 즉시 Git에 있는 값으로 덮어써버립니다. 모든 수정은 Git PR을 통해서만 이루어져야 합니다.
