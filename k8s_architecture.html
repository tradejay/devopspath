K8s 아키텍처 기초

Control Plane과 Worker Node의 조화, 그리고 원하는 상태(Desired State)의 이해

1. Control Plane vs Worker Node

쿠버네티스 클러스터는 크게 명령을 내리는 **Control Plane(두뇌)**과 실제 컨테이너가 돌아가는 **Worker Node(몸체)**로 나뉩니다.

원칙: Desired State (원하는 상태)

"컨테이너 3개를 유지해줘"라고 선언하면, K8s는 누군가 죽더라도 다시 살려서 항상 3개를 유지합니다. 이것이 K8s 운영의 핵심 철학입니다.

2. 핵심 오브젝트 3단계 계층

컨테이너를 효율적으로 관리하기 위해 K8s는 다음과 같은 추상화 계층을 사용합니다.

  • Pod: K8s에서 배포할 수 있는 가장 작은 단위. 하나 이상의 컨테이너를 포함함.
  • ReplicaSet: 지정된 수의 Pod가 항상 실행되도록 보장함.
  • Deployment: Pod와 ReplicaSet의 배포를 관리하며, 롤링 업데이트와 같은 기능을 제공함.
# 아키텍처 확인을 위한 기초 명령어
kubectl get nodes        # 노드 상태 확인
kubectl get pods -A      # 모든 네임스페이스의 포드 확인
kubectl describe node [이름]  # 특정 노드의 상세 정보 확인

주의: Pod를 직접 생성하지 마세요

Deployment 없이 직접 생성한 Pod는 노드에 장애가 발생했을 때 자동으로 복구되지 않습니다. 항상 상위 컨트롤러(Deployment)를 통해 생성해야 합니다.
