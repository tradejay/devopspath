High-Performance Images (이미지 최적화)

작고, 빠르고, 안전한 이미지를 빌드하기 위한 모범 사례

1. 레이어 구조의 이해

도커 이미지는 읽기 전용 레이어들의 쌓임으로 구성됩니다. 각 명령어(RUN, COPY, ADD)는 새로운 레이어를 생성하며, 이 레이어들이 캐싱되어 빌드 속도를 높입니다.

Multi-stage Build: 용량 다이어트의 핵심

컴파일 시 필요한 도구들(Build Dependencies)과 실행 시 필요한 바이너리만 남기는 기법입니다. 수 GB 하던 이미지를 수십 MB로 줄일 수 있습니다.

2. 이미지 최적화 체크리스트

  • Base Image 선택: `alpine`이나 `distroless` 같이 최소한의 패키지만 포함된 이미지를 쓰세요.
  • .dockerignore 활용: 소스 코드 외에 `node_modules`, 로컬 로그, Git 설정 등이 이미지에 포함되지 않게 하세요.
  • 명령어 결합: 레이어 수를 줄이기 위해 연관된 `RUN` 명령어는 `&&`로 묶으세요.
# Multi-stage Dockerfile 예시
FROM node:20-alpine AS builder
WORKDIR /app
COPY package*.json ./
RUN npm install
COPY . .
RUN npm run build

FROM nginx:stable-alpine
COPY --from=builder /app/dist /usr/share/nginx/html
EXPOSE 80
CMD ["nginx", "-g", "daemon off;"]

보안 팁: Root 사용 중지

컨테이너 내부에서도 기본적으로 Root 사용자로 실행하지 않는 것이 좋습니다. `USER` 명령어를 통해 보안이 강화된 특정 유저로 실행되도록 설정하세요.
