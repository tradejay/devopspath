Images & Layers: 빌드 최적화

작고 단단한 이미지를 만드는 Multi-stage 빌드 기법

1. 이미지 레이어 구조

도커 이미지는 여러 개의 읽기 전용 레이어(Layer)가 쌓인 구조입니다. Dockerfile의 각 명령어 하나(RUN, COPY 등)가 새로운 레이어를 형성합니다. 중요한 점은 **레이어 캐싱**입니다. 상단 레이어가 바뀌지 않으면 도커는 이전 빌드 결과를 재사용하여 속도를 높입니다.

2. Multi-stage 빌드의 마법

소스 코드를 빌드할 때는 많은 라이브러리와 툴이 필요하지만, 실행할 때는 결과물(바이너리)만 있으면 됩니다. 이 간극을 메우는 것이 Multi-stage 빌드입니다. 

# 1단계: 빌드 환경 (Build Stage)
FROM node:18 AS builder
WORKDIR /app
COPY . .
RUN npm install && npm run build

# 2단계: 실행 환경 (Production Stage)
FROM node:18-alpine
WORKDIR /app
# 빌드 결과물만 쏙 빼옵니다!
COPY --from=builder /app/dist ./dist
CMD ["node", "dist/main.js"]

3. 실무 팁: .dockerignore

node_modules나 .git 같은 폴더는 이미지에 포함될 필요가 없습니다. .dockerignore 파일을 만들어 불필요한 파일이 레이어에 포함되지 않게 관리하면 이미지 용량을 획기적으로 줄일 수 있습니다.
