Policy as Code (PaC)

문서로 된 보안 규정을 코드로 바꾸어 자동으로 강제하기

1. Policy as Code(PaC)란?

전통적으로 보안 팀은 "S3 버킷은 공개되어선 안 된다"라는 문서를 만들고 나중에 수동으로 검사했습니다. PaC는 이 규정을 코드로 작성하고, 인프라 생성 전/후에 자동으로 검사하여 규정을 준수하지 않는 리소스를 차단하는 기술입니다.

OPA (Open Policy Agent)

정책 의사결정을 위한 오픈소스 표준 엔진입니다. **Rego**라는 언어를 사용하여 "누가 어떤 자원에 접근할 수 있는가", "이 설정이 올바른가"를 판단합니다.

2. PaC의 적용 지점

  • IaC Time: 테라폼 코드를 실행하기 전, 정책 위반이 있는지 사전 검사.
  • Admission Control: Kubernetes에 리소스를 배포하기 전 API 서버 단계에서 거부.
  • Runtime: 이미 실행 중인 시스템이 보안 규정을 벗어났는지 지속적으로 모니터링.
# OPA Rego 코드 예시 (S3 공개 방지)
package aws.s3

deny[msg] {
  resource := input.resource.aws_s3_bucket[_]
  resource.acl == "public-read"
  msg = sprintf("S3 bucket %v is public!", [resource.name])
}

실무 조언: 규정의 자동화

사내 보안 가이드라인이 있다면 그것을 먼저 코드로 옮기는 작업을 시작하세요. 사람이 검토하는 것보다 훨씬 빠르고 정확하며, 개발자들은 배포 전에 문제를 고칠 수 있어 만족도가 높습니다.
