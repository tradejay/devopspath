Policy as Code: 규범 자동화

보안 규정을 코드로 작성하고 시스템이 검사하게 만드는 법

1. Policy as Code (PaC)란?

조직의 보안 가이드라인(예: "모든 S3 버킷은 공용 접근이 금지되어야 한다")을 문서가 아닌 **코드**로 작성하는 것입니다. 사람이 검사하면 빠뜨릴 수 있지만, 코드로 작성하여 CI/CD 파이프라인에 넣어두면 기계가 한 번도 빠짐없이 검사합니다.

2. 핵심 도구: OPA (Open Policy Agent)

OPA는 클라우드뿐 아니라 쿠버네티스, API 테스팅 등 전 분야에서 정책을 검사할 수 있는 표준 도구입니다. **Rego**라는 전용 언어를 사용합니다. 

# Rego 예시: S3 버킷의 퍼블릭 액세스 금지
deny[msg] {
    input.resource_type == "aws_s3_bucket"
    input.public_access_block == false
    msg = "S3 버킷은 반드시 퍼블릭 액세스가 차단되어야 합니다!"
}

3. 인프라 배포 전 가드레일 (Guardrails)

테라폼 코드(`plan`)가 생성되면, 배포하기 전에 OPA가 이 결과를 훑어봅니다. 만약 정책을 위반한 내용이 있다면 배포 버튼 자체가 비활성화되도록 설계하는 것이 성숙한 DevSecOps의 핵심입니다.
