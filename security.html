STEP 06

보안 및 환경 분리 (Security)

가장 취약한 고리를 찾아내고, 인프라를 논리적으로 격리하여 방어 체계 구축하기.

보안은 사후 처리가 아닌 '기본 설계'입니다

데브옵스 환경에서 보안은 개발 속도를 늦추는 장애물이 되어서는 안 됩니다. 소스 코드 수정부터 실제 배포까지 모든 단계에 보안 체크를 자동화하고(DevSecOps), 스테이징과 프로덕션 환경을 철저히 격리하여 장애나 사고의 전파를 막아야 합니다.

이 장에서는 비밀 정보를 관리하는 법, 환경 간 격리 전략, 그리고 실무적인 보안 사고 대응 자동화를 학습합니다.

5단계 전문 학습 로드맵

  • 1. Shift-Left Security

    개발 초기 단계부터 보안 스캐닝(SAST/SCA)을 자동화하여 취약점을 조기에 제거합니다.

  • 2. Secrets Management

    API 키와 비밀번호를 코드에서 분리하고, HashiCorp Vault 등을 이용해 안전하게 관리합니다.

  • 3. Environment Isolation

    네트워크 수준에서의 격리(VPC/Subnet)와 개발/스테이징/리얼 환경의 논리적 분리를 구현합니다.

  • 4. Policy as Code (PaC)

    보안 규정을 코드로 정의하고(OPA), 규정에 맞지 않는 리소스 생성을 자동으로 차단합니다.

  • 5. Incident Automation

    보안 사고 감지 시 알람 발송 및 즉각적인 리소스 차단 프로세스를 실습합니다.
