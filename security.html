STEP 06

보안 및 데이터 환경 격리

개발 중에 유저 데이터를 날리지 않기 위한 필수적 안전 장치 설계.

해야 할 것들 (To-Do)

  • Dev / Stage / Prod 환경변수(.env) 분리

    클라이언트 및 서버 측에서 프로세스 환경 변수를 감지해 개발용 DB와 운영용 DB로의 접속을 철저히 분리합니다.

  • 비밀 키 저장소 사용 (Secret Management)

    깃허브 저장소에 절대 API 키를 올리지 말고, GitHub Secrets나 AWS Parameter Store/Secrets Manager를 연동해 사용합니다.

  • Rate Limiting (경량 DDOS 방어)

    Redis 메모리 캐시를 활용해 비정상적으로 많이 호출되는 악의적인 API 요청 횟수를 제한합니다.

추천 기술 스택

GitHub Secrets AWS Secrets Manager Redis Rate Limiter
