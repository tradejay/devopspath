차세대 패러다임: Data Mesh

데이터 플랫폼의 중앙 집중화를 탈피한 도메인 중심 아키텍처

1. 중앙 데이터 팀의 한계

기존의 '중앙 데이터 레이크' 모델은 모든 데이터 요청이 한 팀으로 몰려 병목(Bottleneck) 현상이 발생합니다. 데이터 엔지니어는 데이터의 의미를 다 이해하지 못한 채 파이프라인만 만들게 됩니다.

Data Mesh: "데이터는 제품이다"

각 비즈니스 도구(도메인) 팀이 자신의 데이터를 직접 소유하고 관리하며, 이를 다른 팀이 쉽게 쓸 수 있도록 '제품(Data Product)'으로서 제공하는 분산화된 방식입니다.

2. 데이터 메쉬의 4대 원칙

  1. 도메인 중심의 소유권: 각 팀이 자기 영역의 데이터를 책임짐.
  2. 제품으로서의 데이터: 데이터를 남이 쓰기 편한 형태로(API, 테이블) 제공.
  3. 자가 서비스 데이터 플랫폼: 공통 인프라 팀은 각 팀이 데이터를 쉽게 처리할 수 있는 도구만 제공.
  4. 연합형 거버넌스: 상호 통신이 가능하도록 최소한의 전사 표준 규칙 적용.

마치며: 데브옵스에서 데이터까지

데이터 메쉬 아키텍처는 마이크로서비스(MSA)가 애플리케이션의 복잡성을 해결한 방식과 매우 닮아 있습니다. 데브옵스 철학이 데이터 영역으로 확장된 최정점의 기술이라고 할 수 있습니다.
