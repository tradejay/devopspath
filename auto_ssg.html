SSG (Static Site Generation)

빌드 타임에 모든 페이지를 미리 만들어 최고의 속도와 보안 얻기

1. 정적 사이트 생성(SSG)의 원리

사용자의 요청이 들어올 때 서버에서 페이지를 만드는(SSR) 것이 아니라, 빌드 시점에 DB나 API로부터 데이터를 가져와 미리 HTML 파일들을 만들어두는 방식입니다.

SSG의 3대 장점

  • 초고속 성능: 서버 연산 없이 미리 생성된 HTML만 서빙하므로 응답 속도가 매우 빠릅니다.
  • 보안성: 데이터베이스나 서버 측 로직이 외부에 노출되지 않아 해킹 위험이 거의 없습니다.
  • 저렴한 비용: 정적 파일 호스팅 시스템(Vercel, GitHub Pages)을 이용하므로 운영 비용이 0에 가깝습니다.

2. 모던 SSG 프레임워크

  • Next.js: 가장 강력한 생태계를 가진 도구. 정적 생성뿐만 아니라 증분 정적 재생성(ISR)을 지원합니다.
  • Astro: 자바스크립트를 최소화하여 극한의 경량화를 추구하는 신흥 강자입니다.
  • Hugo: Go 언어 기반으로 수만 페이지의 빌드도 몇 초 안에 끝내는 압도적인 속도를 자랑합니다.
# Next.js에서 SSG를 위한 코드 예시
export async function getStaticProps() {
  // 빌드 시점에 데이터를 가져옴
  const res = await fetch('https://api.notion.com/...')
  const posts = await res.json()

  return {
    props: { posts },
    revalidate: 3600 // 1시간마다 데이터 갱신 (ISR)
  }
}

전문가의 팁: "Headless" 생각하기

사이트의 외형(Frontend)과 내용(Data)을 완전히 분리하세요. 코드는 GitHub에, 내용은 CMS에 두고 이 둘을 연결하는 자동화 파이프라인을 구축하는 것이 현대적인 블로그의 정석입니다.
