STEP 05 (Requested)

마케팅 & 백데이터 연동 블로그 배포 자동화

서비스 운영 데이터, 주간 통계를 마케팅 블로그 포스트로 변환해 자동 발간하는 파이프라인.

해야 할 것들 (To-Do)

  • 데이터 추출 스크립트 작성 (Python/Node.js)

    Firebase 등 DB에서 지난 일주일 통계 데이터를 추출하여 마크다운(.md) 파일 형태의 텍스트로 가공합니다.

  • 마크다운 파일(Markdown) GitHub 레포지토리 Commit

    스크립트 실행 결과물(.md)을 블로그 소스코드가 있는 저장소의 _posts/ 디렉토리에 Push하는 로직을 작성합니다.

  • GitHub Action Cron 스케줄링

    매주 월요일 아침 9시마다 추출 스크립트가 돌아가도록 GitHub Action 스케줄(Cron) 이벤트를 설정합니다.

  • Next.js Vercel 연동으로 SSG 구현

    Vercel은 포스트가 커밋되면 자동으로 감지해 빌드하므로, Next.js의 정적 생성(Static Site Generation)을 활용하여 빠른 로딩 속도를 달성합니다.
