Container Networking (네트워킹)

컨테이너 간의 통신 방식과 외부 노출 전략 이해하기

1. 주요 네트워크 드라이버

Docker는 목적에 따라 여러 가지 네트워크 모드를 지원합니다.

  • Bridge (기본): 컨테이너끼리만 통신하거나 포트 포워딩을 통해 외부에 노출할 때 사용.
  • Host: 호스트 PC의 네트워크를 직접 사용. 오버헤드가 없지만 포트 충돌 리스크가 있음.
  • None: 네트워크 연결을 차단하여 보안이 극도로 중요한 작업 수행.
  • Overlay: 여러 대의 호스트에 분산된 컨테이너끼리 통신할 때 사용 (Swarm/K8s 기반).

Service Discovery (서비스 검색)

같은 사용자 지정 Bridge 네트워크 내에서는 IP 주소가 아닌 '컨테이너 이름'으로 서로 통신할 수 있습니다. 도커 내부에 내장된 DNS 서버가 이 이름을 자동으로 관리해주기 때문입니다.

2. 포트 매핑 vs 노출

  • EXPOSE: 문서화 용도. 실제로 포트를 열지는 않고 '이 포트를 쓴다'는 정보만 제공.
  • Publish (-p): 호스트의 포트와 컨테이너의 포트를 연결하여 외부 트래픽을 허용함.
# 사용자 정의 네트워크 생성 및 컨테이너 연결
docker network create my-app-net

docker run -d --name db --network my-app-net mysql
docker run -d --name app --network my-app-net my-api-app

보안 팁: 최소 권한 네트워크

모든 컨테이너를 기본 bridge 네트워크에 넣지 마세요. 연관된 서비스끼리만 별도의 네트워크를 만들어 연결하고, 외부 노출이 필요한 서비스만 선택적으로 포트를 Publish 하는 것이 안전합니다.
