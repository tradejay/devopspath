CDN & Asset Optimization

전 세계 어디서든 0.1초 만에 블로그가 로딩되게 만드는 고도화 기법

1. CDN (Content Delivery Network)의 역할

원본 서버(Origin)로부터 데이터를 가져오는 대신, 사용자에게 가장 가까운 위치의 서버(Edge Server)에서 미리 저장된(Cached) 데이터를 전달하는 기술입니다.

Edge Computing

단순한 파일 서빙을 넘어, 사용자에게 가장 가까운 엣지 서버에서 이미지 크기를 줄이거나 인증 처리를 하는 등의 로직을 실행하여 응답 속도를 극한까지 끌어올립니다.

2. 이미지 및 에셋 최적화 전략

  • WebP/AVIF 전환: JPEG/PNG보다 훨씬 작으면서 화질은 좋은 현대적 이미지 포맷을 사용하세요.
  • Lazy Loading: 사용자의 화면에 보이지 않는 이미지는 나중에 로딩하여 초기 로딩 속도를 높입니다.
  • Caching Header: `Cache-Control` 헤더를 통해 브라우저가 정적 파일을 얼마나 오래 보관할지 명시하세요.
# Cloudflare Workers를 이용한 이미지 최적화 예시 (Pseudocode)
addEventListener("fetch", event => {
  const url = new URL(event.request.url)
  // 사용자의 디바이스 환경에 맞춰 이미지 포맷 변경
  if (acceptsWebP(event.request)) {
    url.pathname += ".webp"
  }
  event.respondWith(fetch(url))
})

실무 조언: Cache Purge 자동화

글을 수정했는데 사이트에는 예전 글이 보인다면 캐시가 남아있는 것입니다. CI/CD 파이프라인 마지막에 CDN의 캐시를 강제로 비워주는(Purge) 단계를 추가하여 언제나 최신 상태를 유지하게 하세요.
