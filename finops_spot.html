구매 전략 (Spot/Reserved)

클라우드 결제 모델을 활용한 드라마틱한 비용 절감 기법

1. Spot Instances: 버려지는 자원 빌려쓰기

클라우드 공급자가 남는 리소스를 최대 90% 저렴하게 제공하는 모델입니다. 단, 공급자가 필요할 경우 2분 내에 회수(Termination)할 수 있습니다.

꿀팁: Spot은 개발/배치 작업에 최적!

상태가 없는(Stateless) API 서버나 데이터 처리 작업, CI/CD 빌드 노드 등에 사용하면 비용을 획기적으로 줄일 수 있습니다.

2. 약정 모델 (RI & Savings Plans)

1년 또는 3년 동안 일정량을 쓰겠다고 약속하고 할인을 받는 방식입니다.

  • Reserved Instances (RI): 특정 인스턴스 규격을 선점.
  • Savings Plans: 컴퓨팅 용량($/시간)을 선점. 훨씬 유연함.
# 포트폴리오 제안
- Prod DB (상태 유지): 3년 약정 RI
- Prod API Core: 1년 약정 Savings Plans
- Dev/Staging/Batch: 100% Spot Instances

체크포인트: Spot 회수 대응

K8s의 `node-termination-handler` 등을 사용하여 Spot 회수 알림이 오면 작업을 다른 노드로 안전하게 대피(Drain)시키는 아키텍처를 갖추어야 합니다.
