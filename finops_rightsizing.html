리소스 최적화 (Right-sizing)

지나치게 큰 사양을 줄이고 낭비되는 자원을 원천 차단하기

1. Right-sizing: 딱 맞는 옷 입히기

CPU 사용량이 평소 5%밖에 안 되는데 16코어 인스턴스를 쓰는 것은 낭비입니다. 실제 부하 데이터를 기반으로 적절한 인스턴스 타입으로 변경해야 합니다.

VPA (Vertical Pod Autoscaler)

K8s 환경에서는 VPA가 실제 사용량을 관찰하고 포드의 resource requests 값을 최적의 값으로 추천해줍니다.

2. 좀비 리소스 감사

사용하지 않는데 비용만 나가는 'Ghost Resources'를 주기적으로 정리해야 합니다.

  • Unattached EBS: 인스턴스는 지웠는데 남겨진 디스크.
  • Elastic IP: 연결된 인스턴스가 없는 고정 IP (시간당 과금).
  • Old Snapshots: 수년 전의 백업 데이터.
  • Idle Load Balancers: 타겟 그룹이 비어있는 로드 밸런서.

안티패턴: "나중에 쓸 것 같아요"

클라우드는 필요할 때 즉시 생성할 수 있습니다. 지금 쓰지 않는 리소스는 '일시 중단'이 아니라 '삭제'하는 것이 FinOps의 원칙입니다.
