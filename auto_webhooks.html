Webhooks & Triggers: 이벤트 기반 빌드

콘텐츠 발행 즉시 서버가 스스로 빌드를 시작하는 마법

1. 웹훅(Webhooks)이란?

웹훅은 서비스 A에서 어떤 사건(Event)이 일어났을 때, 서비스 B에게 **"이런 일이 생겼어!"**라고 신호를 보내는 방식입니다. Jamstack 아키텍처에서는 CMS에서 '발행' 버튼을 눌렀을 때 Vercel이나 Netlify 같은 배포 서버에 신호를 보낼 때 사용됩니다.

2. 자동 빌드 Flow 설계

  1. 마케터가 CMS에서 블로그 포스트를 작성하고 **[Publish]** 버튼 클릭.
  2. CMS는 미리 등록된 **Build Hook URL**로 POST 요청을 보냄.
  3. 배포 서버(Vercel)는 요청을 받는 즉시 최신 콘텐츠를 API로 긁어옴.
  4. 정적 사이트를 새로 빌드하여 전 세계에 배포.
# Vercel Build Hook 예시
https://api.vercel.com/v1/integrations/deploy/prj_abcd...
# 이 URL을 CMS의 Webhook 설정 칸에 붙여넣기만 하면 연결 끝!

3. 실무 팁: 빌드 최적화 (ISR)

페이지가 수만 개라면 매번 전체를 빌드하는 것은 비효율적입니다. **증분 정적 재생성(ISR)** 기법을 사용하면 바뀐 페이지만 쏙쏙 골라 업데이트하여 사용자 대기 시간을 최소화할 수 있습니다.
