Multi-Container: Docker Compose

여러 컨테이너를 하나로 묶어 관리하는 오케스트레이션 기초

1. 왜 Docker Compose인가?

현대적인 서비스는 웹 서버, DB, 레디스, 워커 등 여러 개의 컨테이너가 협력해서 동작합니다. 이것을 매번 docker run으로 수동 실행하는 것은 불가능에 가깝습니다. Docker Compose는 이 복잡한 구성을 한 장의 docker-compose.yml 선언서로 관리하게 해줍니다. 

version: '3.8'
services:
  web:
    build: .
    ports:
      - "80:3000"
    depends_on:
      - db
  db:
    image: postgres:15
    volumes:
      - db_data:/var/lib/postgresql/data
volumes:
  db_data:

2. 멱등성 (Idempotency)의 즐거움

docker-compose up -d 명령어를 실행하면 도커는 현재 상태와 파일에 적힌 희망하는 상태를 비교합니다. 이미 실행 중인 컨테이너는 건드리지 않고, 바뀐 부분만 업데이트하거나 새로 만듭니다. 이러한 **선언적 방식**은 데브옵스 철학의 핵심인 '코드형 인프라'의 맛보기라고 할 수 있습니다.

3. 실무 팁: 개발 환경의 통일

팀원 모두가 똑같은 docker-compose.yml을 가지고 있으면 "내 컴퓨터에 DB가 안 깔려 있어요"라는 말이 사라집니다. 한 번의 명령으로 모든 팀원이 동일한 개발 환경을 100% 재현할 수 있게 되는 것이 프로젝트 자동화의 종착역입니다.
