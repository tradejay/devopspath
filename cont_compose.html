Multi-Service with Compose

복잡한 서비스 환경을 한 번에 제어하는 선언형 선언문 작성

1. 왜 Docker Compose인가?

실제 애플리케이션은 단일 컨테이너가 아니라 Web, API, Database, Redis 등 여러 서비스가 연결된 구조입니다. 이를 수동으로 하나씩 띄우는 것은 매우 비효율적입니다.

Docker Compose의 장점

  • 선언적 정의: `docker-compose.yml` 파일 하나로 인프라를 정의합니다.
  • 독립된 네트워크: 서비스끼리만 통신할 수 있는 격리된 네트워크를 자동으로 구축합니다.
  • 환경 재현: 어떤 PC에서든 `docker-compose up` 명령 하나로 동일한 개발 환경을 구축합니다.

2. 서비스 의존성 및 환경 설정

서비스 간의 선후 관계(`depends_on`)와 외부로 노출하지 않을 내부 변수 관리가 핵심입니다.

# docker-compose.yml 예시
services:
  web:
    build: .
    ports:
      - "80:80"
    depends_on:
      - db
  db:
    image: postgres:15-alpine
    environment:
      POSTGRES_PASSWORD: ${DB_PASSWORD}
    volumes:
      - db_data:/var/lib/postgresql/data

volumes:
  db_data:

운영 팁: 환경 변수 관리

비밀번호나 API 키 같은 민감 정보는 YAML에 직접 노출하지 마세요. `.env` 파일을 활용하거나 CI/CD 도구의 Secrets 기능을 연결하여 유연하게 대입해야 합니다.
